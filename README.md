# 知忆 MemoBase

基于混合检索、记忆管理与 ReAct 编排的知识库智能体平台设计与实现。

`知忆（MemoBase）` 是一个面向课程团队、实验室和小型组织知识管理场景的知识库智能体系统。项目以 `Go` 作为后端开发语言，以 `React` 构建 Web 前端，结合 `Milvus` 向量数据库、`BM25` 关键词检索、外置与本地大模型接入、记忆管理和 `ReAct` 智能体编排能力，提供一个兼具实用价值与工程展示价值的智能知识平台。

## 项目定位

本项目是一个围绕“知识接入、混合检索、记忆利用、智能体执行、工程化部署”构建的知识库智能体平台。

目标场景包括：

- 课程资料整理与问答
- 实验室文档沉淀与检索
- 小型团队内部知识管理
- 项目过程资料汇总、总结与辅助分析

## 核心能力

- 知识库管理：支持知识库创建、维护和文档归档
- 文档处理：支持文档上传、解析、切片与索引构建
- 混合检索：结合 `BM25` 与 `Milvus` 向量检索提升召回效果
- 智能问答：基于知识库进行问答，并展示引用来源
- 记忆管理：支持短期记忆与长期记忆的提取、存储和召回
- 智能体编排：基于 `ReAct` 执行知识检索、记忆检索和上下文整理
- 模型网关：统一接入外置大模型与本地模型
- 工程化部署：支持 Docker、`Kubernetes`、`Prometheus`、`Grafana`

## 最小可行产品（MVP）

第一阶段目标是交付一个可演示的最小可行版本，核心闭环如下：

`用户上传文档 -> 系统构建 BM25 与向量索引 -> 用户发起提问 -> 智能体调用知识检索与记忆检索 -> 模型生成回答 -> 前端展示答案与引用 -> 监控面板展示运行指标`

MVP 计划包括：

- 用户登录与退出
- 知识库创建、查看、删除
- 文档上传、解析、切片
- `BM25` 索引与 `Milvus` 向量索引构建
- 基于混合检索的问答
- 基础记忆管理
- 单个知识库助手智能体
- 外置模型与本地模型切换
- Docker 部署与基础监控面板

## 最终功能展望

项目完整版本将逐步扩展以下能力：

- 多知识库组织与知识运营
- 更完整的会话与任务历史中心
- 更稳定的记忆提取与记忆召回机制
- 更丰富的智能体工具与任务型执行能力
- 更完善的部署、监控和运行维护体系

## 系统架构概览

系统计划采用前后端分离架构，并按以下层次组织：

- 前端层：`React` Web 界面
- API 层：`Go` 后端服务
- 检索层：`BM25 + Milvus` 混合检索
- 记忆层：短期记忆与长期记忆管理
- 智能体层：基于 `ReAct` 的工具调用编排
- 模型层：外置模型与本地模型统一网关
- 运维层：Docker、`Kubernetes`、`Prometheus`、`Grafana`
